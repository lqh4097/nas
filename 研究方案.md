# 面向边缘硬件感知的多目标进化神经架构搜索研究方案

## 核心问题

RK3566 边缘端算力严重受限（CPU: 4×Cortex-A55, NPU: 0.8 TOPS），如何在精度和推理延迟之间自动找到最优平衡，打破手工架构的瓶颈。

## 量化目标

| 目标 | 最低要求 | 理想值 |
|---|---|---|
| 鸟鸣分类 Top-1 Accuracy | > 80% | > 90% |
| RK3566 单次推理延迟 | < 50ms | < 20ms |
| 模型参数量 | < 2M | < 500K |
| 模型存储大小（INT8量化后） | < 5MB | < 1MB |

| 项目 | 说明 |
|------|------|

| 标注方式 | 按文件夹结构确定类别（ImageFolder 兼容）|
| 类别不均衡比 | 约 2:1（最多类 820 张 vs 最少类 408 张原始）|

**音频增强方法**（在原始音频上操作后重新生成 mel 图）：

| 增强方法 | 模拟场景 |
|---------|---------|
| PitchShift | 不同个体音高差异 |
| AddColoredNoise | 风声/背景噪声干扰 |
| PeakNormalization | 不同录音设备/距离的音量差异 |
| BandStopFilter | 特定频段被遮挡或设备频响不平坦 |

每条原始录音生成 4 个增强变体，增强比 ×5（原始 1 + 增强 4）。

**数据划分**（无需移动文件，按索引区分）：

| 集合 | 来源 | 样本数 | 说明 |
|------|------|--------|------|
| train | Mel_Augment_four/train | 132,230 | 含原始 + 4 种增强，用于模型训练 |
| val | Mel_Augment_four/val 随机 50% | 3,306 | 纯原始 mel，训练监控 + NAS 搜索评估 |
| test | Mel_Augment_four/val 剩余 50% | 3,306 | 纯原始 mel，封存，仅用于最终论文指标 |

val/test 均来自纯原始录音（非 train 增强图的衍生），无数据泄漏。
test 共 3,306 张，每类约 83 张，95% CI 宽度约 ±1.5%，满足论文汇报要求。
val/test 索引由固定随机种子（seed=42）生成，保存至 `data/split_index.json`。

### 数据预处理 pipeline

```
Mel_Augment_four/train（val/test）目录下的 PNG 文件
```

## 搜索空间定义（核心）

### 宏观架构编码

每个个体用一个整数向量表示，共 16 维：

```
编码位置  参数              取值范围            #取值
──────────────────────────────────────────────────
 0      stem_channels      [16, 24, 32]         3
 1      n_stages           [2, 3]              2
 2      stem_stride        [1, 2]               2
 3      n_layers@stage0    [2, 3, 4]            3
 4      n_layers@stage1    [2, 3, 4]            3
 5      n_layers@stage2    [2, 3, 4]            3
 6      kernel@stage0      [3, 5]               2
 7      kernel@stage1      [3, 5]               2
 8      kernel@stage2      [3, 5]               2
 9      expand@stage0      [1, 2, 3, 4]         4
10      expand@stage1      [1, 2, 3, 4]         4
11      expand@stage2      [1, 2, 3, 4]         4
12      use_se@stage0      [0, 1]               2
13      use_se@stage1      [0, 1]               2
14      use_se@stage2      [0, 1]               2
15      activation         [0(relu), 1(hswish)] 2

总搜索空间大小（计入 n_stages 门控；每 stage 基因组合 = n_layers·kernel·expand·se = 3·2·4·2 = 48）:
  n_stages=3: 3(stem)×2(stride)×2(act)×48³ = 1,327,104
  n_stages=2: 3×2×2×48²                    =    27,648
  合计 ≈ 1,354,752 ≈ 135 万
```

### 微观构建块（每个 stage 内的搜索单元）

```python
class SearchBlock:
    """倒残差 + 可选 SE + 可选跳跃连接，MobileNetV2 block 的搜索版本"""
    def __init__(in_ch, out_ch, kernel, expand, use_se, act, stride):
        # 1. 1×1 expand（如果 expand > 1）
        # 2. depthwise conv (kernel × kernel)
        # 3. SE (如果 use_se == 1)
        # 4. 1×1 project (恢复到 out_ch)
        # 5. skip connection (如果 in_ch == out_ch and stride == 1)

# 输入规格（固定，不在搜索空间内）：
# in_channels = 3（RGB），input_size = 224×224
# stem: Conv2d(3, stem_channels, 3×3, stride=stem_stride)
```

### 搜索空间设计理由（面向时频稀疏信号）

```
设计决策                          针对的声学特性
─────────────────────────────────────────────────
kernel 可选 3 或 5                鸟鸣音节需要更大感受野捕获频率邻域
expand 可选 1-4                  时频稀疏 → 不需过宽的中间表示
SE 可选                           不同鸟种的关键判别频带不同（注意力）
skip connection 有条件            保持浅层时频细节的传递
stem_stride 可选 1 或 2           控制早期下采样程度（保留高频细节）
```

### 算子级硬件成本表

为搜索空间中每个算子预定义硬件成本权重 $C(op)$，作为先验引导信号（不替代实测延迟）。

| 算子类别 | 具体算子/选择 | 成本 C(op) | 理由 |
|---------|-------------|-----------|------|
| **轻量卷积** | 3×3 depthwise | 1 | RK3566 NPU 对 3×3 dw 卷积有专项优化 |
| **中等卷积** | 5×5 depthwise | 2 | FLOPs 约为 3×3 的 2.8× |
| **标准卷积** | 1×1 pointwise | 1 | NPU 最高效的算子 |
| **激活函数** | ReLU | 1 | 几乎无硬件开销 |
| | HardSwish | 2 | 比 ReLU 多 ~30% 延迟 |
| **注意力模块** | SE (use_se=1) | 3 | GlobalAvgPool + 2×FC，额外归约操作 |
| **扩展比** | expand=1/2/3/4 | 1/2/3/4 | 通道数线性增加，memory bandwidth 压力 |
| **层数深度** | n_layers=2/3/4 | 1/2/3 | 深度线性增加 |
| **stem 配置** | stem_channels=16/24/32 | 1/2/3 | 输入通道数 |
| **下采样** | stride=1/2 | 1/2 | stride=2 后续层 FLOPs 指数降低（正向） |

**架构级硬件成本聚合**：

$$C_{total}(A) = \sum_{i \in \text{stages}} \left( C(kernel_i) + C(expand_i) + C(use\_se_i) + C(n\_layers_i) \right) + C(stem\_channels) + C(activation)$$

用途：① 种群初始化偏置；② 变异方向偏置；③ 前沿可视化辅助轴。

> 成本表为手工先验，不替代实测。阶段2 延迟数据集到位后，用 Spearman 相关校验 $C_{total}$ 与 RK3566 实测延迟的一致性（目标 > 0.8）；相关性结果写入论文作为成本表合理性的证据，必要时按实测回标各算子权重。

## 基线模型配置

> **对比框架（重要）**：本任务准确率已接近饱和（实测 Manual-CNN 98.4%、MobileNetV2 98.8%），
> 几乎所有基线都会落在 97–99%，**纯准确率对比无区分度**。故基线对比的核心放在**效率轴**：
> 在 Acc ≥ 97% 的等准确率约束下，比较 RK3566 实测延迟、参数量、FLOPs。
> 搜索空间参数量实测范围 0.015–3.66M（中位 0.20M），下列基线预算均落在可达区间内。
>
> **统一口径**：所有基线一律同一管线**从零训练**（SpecAugment、相同 epoch、≥3 seed 取均值），
> 不使用 ImageNet 预训练，以保证与 NAS 搜出的架构同口径可比。

### 核心基线（必做）

| 类别 | 模型 | 参数量 | 作用 |
|------|------|--------|------|
| 手工下限 | Manual-CNN | ~0.5M | 4层CNN+2层FC，sanity 下限：搜索结果应在效率轴上支配它 |
| 同范式关键 | MobileNetV2 0.5× | ~1.4M | **最重要基线**：搜索空间即 MobileNetV2 范式，直接"手工 vs 搜索"同范式对照 |
| 异范式 | ShuffleNetV2 0.5× | ~1.4M | Channel shuffle 范式，验证跨范式优势 |
| 精度上限参照 | MobileNetV3-Small | ~2.5M | 行业标准，作"大模型精度天花板"参照（NAS 以 ~1/5 参数逼近其精度即为成功）|

### 搜索方法基线（必做）

| 基线 | 说明 |
|------|------|
| Random Search ×200 | 随机采样 200 个取最优，验证进化引导的必要性 |
| Standard NSGA-II | 即消融的 `baseline` 组（关 EDA + Surrogate + 智能初始化），验证三组件的整体增益 |

### 选做基线（视时间/价值取舍）

| 模型 | 参数量 | 取舍理由 |
|------|--------|---------|
| Manual-CRNN（CNN+GRU）| ~0.3M | 佐证"边缘部署不该用 RNN"。⚠️ **GRU 在 RK3566 NPU 常不支持、回退 CPU，延迟差**——这恰是支持纯 CNN 搜索空间的论据，但须在论文显式标注该部署 caveat，否则其延迟会"莫名其妙地差" |
| GhostNet 0.5× | ~1.3M | 异范式补充；饱和任务下边际价值低，时间紧可略 |
| EfficientNet-Lite0 | ~2M | **替代原 EfficientNet-B0 0.5×**：B0 本身是 NAS 产物（拿它当基线混淆"你的搜索 vs 它的搜索"）、无标准 0.5× 缩放、torchvision 不直供；改用专为边缘设计的 Lite0 更干净，或直接删 |

## 实验设计 —— 围绕 4 个 Research Questions 组织

### RQ1：搜索出的架构在「精度–效率」权衡上是否优于手工设计基线？

> 本任务准确率饱和（基线普遍 97–99%），故 RQ1 **不以"准确率更高"为主张**，
> 而以**等准确率下效率更优 / Pareto 支配**为主张。

| 对比项 | 方法 | 报告指标 |
|---|---|---|
| 核心手工基线 | Manual-CNN, MobileNetV2 0.5×, ShuffleNetV2 0.5×, MobileNetV3-Small（选做：Manual-CRNN, GhostNet 0.5×, EfficientNet-Lite0）| Top-1 Acc, RK3566 实测 Latency, Params, FLOPs |
| 随机搜索 | 200 个随机架构，报告最优 5 个的均值±std | 同上 + HV vs 评估预算曲线 |
| Ours（Full：EDA+Surrogate+分层初始化） | 50 代 × 40 个体，独立重复 5 次 | 同上 + Pareto 前沿图 |

**主结果图**：Acc–Latency 散点 + Pareto 前沿——手工基线为散点、Ours 为前沿曲线，
展示"同精度下 Ours 延迟/参数更低（基线点被前沿支配）"。
**判定标准**：① Ours 前沿支配各手工基线点；② 在 Acc ≥ 97% 处，Ours 最低延迟 < 最优基线延迟。

> RQ1 最终上报的 Acc/Latency 必须是**完整训练 + RK3566 上板实测**值，不能用 surrogate 预测值——surrogate 仅用于搜索过程加速。

### RQ2：发现的架构有什么结构特征？

分析方法：
1. **架构可视化**：从前沿均匀采样 4-6 个代表架构，绘制计算图
2. **编码频率分析**：统计前沿个体的 kernel/SE/expand/n_layers 分布
3. **算子成本分布**：accuracy-latency-$C_{total}$ 三轴关系，验证搜索是否自然偏好低成本结构

### RQ3：各改进组件的贡献分别是什么？（消融实验）

采用**从 Full 留一法**，每组只相对 Full 移除一个组件，干净归因各组件的边际贡献（避免「+EDA 同时含 Init」式的混淆）：

| 实验组 | 初始化 | 评估 | 生成 | 移除的组件 |
|--------|------|------|------|------|
| **Full** | 分层+成本引导 | 代理评估+定期校准 | EDA 概率采样 | —（完整方案）|
| Full−Init | 随机初始化 | 代理评估+定期校准 | EDA 概率采样 | 硬件感知初始化 |
| Full−Surrogate | 分层+成本引导 | 短训(20ep)评估 | EDA 概率采样 | 代理模型 |
| Full−EDA | 分层+成本引导 | 代理评估+定期校准 | 交叉+变异 | EDA 生成 |
| Baseline | 随机初始化 | 短训(20ep)评估 | 交叉+变异 | 全部（标准 NSGA-II）|

- **算力压缩**：搜索阶段 pop=20、gen=15、独立重复 5 次（重复数保 5 以满足 Wilcoxon 显著性下限）；非代理组的「评估」用 20-epoch 短训而非 50-epoch 完整训练。
- **公平计分**：HV 与最终上报指标统一用各组**最终前沿的真实指标**（完整 50-epoch 训练 + RK3566 实测延迟）计算，避免不同评估器导致口径不一致。
- 报告：最终 HV、HV vs 代数曲线、HV 箱线图、Wilcoxon 组间检验。

### RQ5（选做）：Post-NAS 结构化剪枝对最优架构的进一步压缩

NAS 决定宏观架构（层数、宽度、算子），剪枝在此基础上删除训练后的冗余权重，属于不同阶段的正交压缩手段，可叠加使用。

**动机**：NuWa（CVPR 2025）在 Jetson Orin NX（同类 ARM 边缘板）上实验证明，INT8 量化因缺乏 TensorRT 支持吞吐量下降 98%+，非结构化稀疏剪枝几乎无加速——结构化剪枝是边缘部署唯一实用的后处理压缩手段。NAS 搜出的最优架构若参数量仍偏高（1M–2M），可通过剪枝进一步逼近 <500K 的理想目标。

**方法**：对 NAS 最优架构的 pointwise（1×1）卷积层执行激活引导结构化剪枝：

$$W_2' = W_2 \mathcal{H}^\top \mathcal{H}_r (\mathcal{H}_r^\top \mathcal{H}_r)^\dagger$$

其中 $\mathcal{H}_r$ 为激活均值最高的 $e'$ 个神经元对应的特征矩阵（128 张校准样本），闭合最小二乘解重建剩余权重，**无需 retrain**。

**实验设计**：

| 对比组 | 说明 | 报告指标 |
|---|---|---|
| NAS-best | Pareto 前沿最优架构（未剪枝） | Acc, Latency, Params |
| NAS-best + Pruning (α=0.2) | 剪枝率 20%，无重训 | 同上 + 精度损失 |
| NAS-best + Pruning (α=0.4) | 剪枝率 40%，无重训 | 同上 + 精度损失 |
| NAS-best + Pruning + Fine-tune | 剪枝后 10 epoch 微调 | 同上 |

若 RQ4 蒸馏已做，则额外对比：NAS → 蒸馏 → 剪枝 的三段式流程（蒸馏提精度上限，剪枝再降体积）。

> 条件不具备时可推迟，不影响论文主体。开销：剪枝本身仅需分钟级（闭合解），Fine-tune 约 1–2 GPU·小时。

## 算法改进：EDA + Surrogate 融合搜索

### 与最接近工作的区别（PMB-CAS）

最接近的工作是 PMB-CAS（Sun et al., "Multi-Objective Neural Architecture Search for Cognitive Diagnosis," IEEE TCSS, 2026）。本方案的 EDA 概率建模、NSGA-II 选精英、算子加权成本表、温度退火采样、Laplace 平滑、4-RQ 框架均借鉴该范式——**论文中相关公式（温度采样、条件概率）与框架须显式引用 PMB-CAS**，否则与原文雷同会被审稿人识别。在此之上，本方案可主张的贡献（delta）：

| 贡献 | 相对 PMB-CAS 的区别 |
|---|---|
| ① 新问题域 + 真机硬件目标 | PMB-CAS 是表格认知诊断，第二目标是从架构免费算出的拓扑可解释性；本方案是边缘鸟鸣识别，第二目标为 **RK3566 真机实测延迟**（需 RKNN-Toolkit2 上板 + 延迟预测器），属硬件感知 NAS 这一不同问题类 |
| ② EDA + 双 Surrogate 融合 | PMB-CAS 明确**不用** surrogate，将 EDA 与代理辅助视为对立的两条路；本方案二者合一——Surrogate 加速评估 + EDA 引导生成 |
| ③ 定长编码的架构层级条件 EDA | PMB-CAS 对变长计算树建树文法 P(type\|parent,pos,depth)；本方案在定长 16 维类别向量上，依架构层级语义（n_stages 门控 + stage 内容量耦合）自定义依赖结构（见组件二）|
| ④ 硬件实测校准的成本表 | PMB-CAS 的 C(op) 是主观可解释性权重、无物理校准；本方案 C(op) 用真机延迟回标（Spearman > 0.8）|

> 定位：**应用新域 + 范式融合**，不主张"全新算法"。过度宣称 EDA 本身的新颖性会因与 PMB-CAS 雷同而被拒。

### 动机：标准 NSGA-II 的两个瓶颈

```
瓶颈 1：评估太慢
  → 每代 40 个体 × 完整训练 50 epoch = 极其昂贵
  → 解决：Surrogate 代理模型替代完整训练

瓶颈 2：生成太盲
  → 交叉+变异是「无记忆」的随机扰动，好的结构模式容易被破坏
  → 解决：EDA 从精英种群学习分布，定向采样
```

### 整体流水线

```
                 ┌─── 代理层 1：评估（Surrogate）───┐
                 │  精度预测器（学习曲线外推,~1/5训练）│
                 │  延迟预测器（纯编码→延迟, 毫秒级） │
                 └──────────────────────────────────┘
                         ↓ 预测值送入
              ┌──────────────────────┐
              │   NSGA-II 非支配排序   │
              │   + 拥挤距离选择精英   │
              └──────────────────────┘
                         ↓ 精英子集 P_elite
                 ┌─── 代理层 2：生成（EDA）─────────┐
                 │  从 P_elite 学习概率分布 M_prob    │
                 │  → 温度退火控制探索/利用            │
                 │  采样 N_sample 个新个体             │
                 │  + N_random 随机注入保持多样性      │
                 └──────────────────────────────────┘
                         ↓ 形成下一代种群
              ┌──────────────────────┐
              │   定期校准（每 k 代）  │
              │   真实评估 5 个精英    │
              │   → 增量更新 Surrogate │
              └──────────────────────┘
```

### 算法伪代码

```
Algorithm: EDA-Surrogate NSGA-II for Edge NAS

输入: 搜索空间 S, 种群大小 N, 代数 T_max, 校准间隔 k
输出: 最终 Pareto 前沿

1. 离线: 随机采样 300 个架构，完整训练 → 训练初始 Surrogate
2. 初始化种群 P_0: 硬件感知分层初始化 (N 个个体)
3. for t = 0 to T_max - 1:
4.     // ① 快速评估（Surrogate）
5.     F(A) = ( -SurrAcc(A), SurrLat(A) )   for each A ∈ P_t
6.     // ② 精英选择（NSGA-II）
7.     P_elite = NonDominatedSort + CrowdingDistance
8.     // ③ 概率建模（EDA）
9.     M_prob = BuildProbModel(P_elite)   // 学习架构层级条件分布（见组件二）
10.    // ④ 温度控制采样
11.    τ = max(0.1, 1.0 - 0.9 × t / T_max)
12.    P_{t+1} = SampleFromModel(M_prob, τ, N_sample) + RandomInit(N_random)
13.    // ⑤ 定期校准
14.    if t % k == 0:
15.        真实评估前沿 top-5 → 加入校准集 → 增量更新 Surrogate
```

### 组件一：Surrogate 代理模型

**离线数据**：随机采样 300 个架构完整训练，得到 `{(编码向量, 真实Acc, 真实Lat)}`。

**精度预测器**：输入 = 16 维编码 + 早停 5-epoch accuracy → 预测最终 accuracy（XGBoost 或 2 层 MLP）。

**延迟预测器**：输入 = 16 维编码 → 预测 RK3566 推理延迟（线性回归或轻量 MLP，预期 R² > 0.90）。

**评估路径**：
```
编码向量 → 解码为 PyTorch 模型 → 训练 5 epoch（早停）
    ├→ 早停 acc + 编码 → 精度预测器 → 预测 accuracy
    └→ 编码 → 延迟预测器 → 预测 latency
总耗时：约完整训练的 1/5
```

### 组件二：EDA 概率建模（架构层级条件分布）

> 概率建模范式沿用 PMB-CAS，但 PMB-CAS 面向变长计算树、建模树文法 P(type|parent,pos,depth)；本方案编码为**定长 16 维类别向量**，无法套用树文法，故依据**编码的架构层级语义**自定义依赖结构。这是相对 PMB-CAS 的算法 delta（贡献③）。

将 16 维分两层建模：

**① 全局基因（边缘分布）**：stem_channels、n_stages、stem_stride、activation 各自独立学习边缘分布 $P(x_i)$。

**② 每 stage 基因（门控 + stage 内联合）**：n_stages 为门控位，决定哪些 stage 生效；对每个**生效**的 stage $s$（$s <$ n_stages），联合建模其 (kernel_s, expand_s, use_se_s, n_layers_s)，条件于 (n_stages, stage 序号 $s$)——捕获 stage 内 4 基因的容量耦合（如高 expand 常伴浅层以控成本）。

条件概率 + Laplace 平滑（α=1.0），$c$ 为上述条件上下文（全局基因 $c=\varnothing$；stage 基因 $c=$(n_stages, $s$)）：

$$P(op_i | c) = \frac{\text{count}(op_i, c) + \alpha}{\sum_j \text{count}(op_j, c) + |\mathcal{O}| \cdot \alpha}$$

**采样顺序**：先采全局基因 → 定下 n_stages → 逐 stage 按联合分布采样生效 stage 的基因（非生效 stage 置空）。

温度控制采样（沿用 PMB-CAS，须引用）：

$$p_i' = \frac{p_i^{1/\tau}}{\sum_j p_j^{1/\tau}}, \quad \tau(t) = \max(0.1,\ 1.0 - 0.9 \times t / T_{max})$$

### 组件三：硬件感知种群初始化

初始种群分三块，每块用不同概率偏置生成：

1. **轻量子种群**（1/3）：偏小 stem_channels、低 expand ratio、kernel=3、浅层
2. **重量子种群**（1/3）：偏大 stem_channels、高 expand ratio、深层
3. **中等子种群**（1/3）：完全随机，填补中间地带

### 改进效果预期

| 维度 | 标准 NSGA-II | +Surrogate | +EDA | **Full** |
|------|-------------|------------|------|----------|
| 每代评估耗时 | ~10h | ~2h | ~10h | **~2h** |
| 48h 内可探索代数 | ~5 代 | ~24 代 | ~5 代 | **~24 代** |
| 生成方向性 | 无 | 无 | 有 | **有** |
| 精英模式保留 | 可能破坏 | 可能破坏 | 保留 | **保留** |
