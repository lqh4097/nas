# 面向边缘硬件感知的多目标进化神经架构搜索研究方案

## 核心问题

RK3566 边缘端算力严重受限（CPU: 4×Cortex-A55, NPU: 0.8 TOPS），如何在精度和推理延迟之间自动找到最优平衡，打破手工架构的瓶颈。

## 量化目标

| 目标 | 最低要求 | 理想值 |
|---|---|---|
| 鸟鸣分类 Top-1 Accuracy | > 80% | > 90% |
| RK3566 单次推理延迟 | < 50ms | < 20ms |
| 模型参数量 | < 2M | < 500K |
| 模型存储大小（INT8量化后） | < 5MB | < 1MB |

## 数据集

实验室自建鸟鸣数据集（Mel_Augment_four，40 类）：

| 项目 | 说明 |
|------|------|
| 鸟种数 | 40 种，每种一个文件夹（Class 0–39）|
| 原始样本数 | 每类约 408–820 张，共 26,446 张 |
| 增强后样本数 | 每类约 2,040–4,100 张，共 132,230 张（train）|
| 数据格式 | PNG mel-spectrogram，224×224 RGBA 伪彩色（viridis）|
| 标注方式 | 按文件夹结构确定类别（ImageFolder 兼容）|
| 类别不均衡比 | 约 2:1（最多类 820 张 vs 最少类 408 张原始）|

**音频增强方法**（在原始音频上操作后重新生成 mel 图）：

| 增强方法 | 模拟场景 |
|---------|---------|
| PitchShift | 不同个体音高差异 |
| AddColoredNoise | 风声/背景噪声干扰 |
| PeakNormalization | 不同录音设备/距离的音量差异 |
| BandStopFilter | 特定频段被遮挡或设备频响不平坦 |

每条原始录音生成 4 个增强变体，增强比 ×5（原始 1 + 增强 4）。

**数据划分**（无需移动文件，按索引区分）：

| 集合 | 来源 | 样本数 | 说明 |
|------|------|--------|------|
| train | Mel_Augment_four/train | 132,230 | 含原始 + 4 种增强，用于模型训练 |
| val | Mel_Augment_four/val 随机 50% | 3,306 | 纯原始 mel，训练监控 + NAS 搜索评估 |
| test | Mel_Augment_four/val 剩余 50% | 3,306 | 纯原始 mel，封存，仅用于最终论文指标 |

val/test 均来自纯原始录音（非 train 增强图的衍生），无数据泄漏。
test 共 3,306 张，每类约 83 张，95% CI 宽度约 ±1.5%，满足论文汇报要求。
val/test 索引由固定随机种子（seed=42）生成，保存至 `data/split_index.json`。

### 数据预处理 pipeline

## 搜索空间定义（核心）

### 宏观架构编码

每个个体用一个整数向量表示，共 17 维（1 个全局门控 + 3 stage × 5 基因 + 1 输入分辨率）：

```
编码位置  参数                取值范围                       #取值
──────────────────────────────────────────────────────────────────
 0      n_stages            [2, 3]                          2    ← 门控位
 1      n_blocks@stage1     [1, 2, 3]                       3
 2      channels@stage1     [16, 24, 32, 48]                4
 3      kernel@stage1       [3, 5]                          2
 4      expand@stage1       [3, 6]                           2
 5      use_se@stage1       [0, 1]                           2
 6–10   stage2 五基因        同上（channels [32,48,64,96]）
11–15   stage3 五基因        同上（channels [64,96,128,192]）
 16     resolution          [64,96,128,160,192,224]         6    ← 输入分辨率协同搜索

每 stage 基因组合 = n_blocks·channels·kernel·expand·se = 3·4·2·2·2 = 96

编码空间大小（含 n_stages 门控造成的冗余）:
  2(n_stages) × 96³ × 6(分辨率) = 10,616,832 ≈ 1062 万

去重后的真实不同架构数（n_stages=2 时 stage3 的 5 基因被 decode 忽略）× 6 分辨率:
  (96³ + 96²) × 6 = 893,952 × 6 = 5,363,712 ≈ 536 万
```

> 论文表述建议：**约 536 万个有效架构（编码空间 1062 万）**；其中输入分辨率与网络结构**协同搜索**（见组件三补充与"搜索空间局限"）。

### 微观构建块（每个 stage 内的搜索单元）

```python
class InvertedResidual:
    """MobileNetV2 倒残差块 + 可选 SE（MobileNetV3）+ 可选残差"""
    def __init__(in_ch, out_ch, kernel, expand, use_se, stride):
        # 1. 1×1 expand（hidden = in_ch × expand，expand ∈ {3,6}）
        # 2. depthwise conv (kernel × kernel)，kernel ∈ {3,5}
        # 3. SE（如果 use_se == 1，Hardsigmoid 门控）
        # 4. 1×1 project（恢复到 out_ch）
        # 5. 残差（如果 in_ch == out_ch and stride == 1）
        # 激活：ReLU6

# 固定（不在搜索空间内）：
# in_channels=3（RGB），input_size=224×224
# stem: Conv2d(3, 16, 3×3, stride=2)（stem 通道与 stride 固定）
# activation: ReLU6（不搜索）
# 每 stage 第一个块 stride=2 下采样，其余 stride=1
```

### 搜索空间设计理由（面向时频稀疏信号）

```
设计决策                          针对的声学特性
─────────────────────────────────────────────────
kernel 可选 3 或 5                鸟鸣音节需要更大感受野捕获频率邻域
expand 可选 3 或 6               时频稀疏 → 不需过宽的中间表示
channels 每 stage 4 档            按 stage 递增，控制各层容量
SE 可选                           不同鸟种的关键判别频带不同（注意力）
n_blocks 可选 1-3                控制每 stage 深度
n_stages 可选 2 或 3             门控整体深度
```

### 算子级硬件成本表

为搜索空间中每个算子预定义硬件成本权重 $C(op)$，作为先验引导信号（不替代实测延迟）。

为搜索空间中**可搜索的**每个算子/选择预定义成本权重 $C(op)$（stem、激活已固定，不在表内）。

| 算子类别 | 具体算子/选择 | 成本 C(op) | 理由 |
|---------|-------------|-----------|------|
| **轻量卷积** | 3×3 depthwise | 1 | RK3566 NPU 对 3×3 dw 卷积有专项优化 |
| **中等卷积** | 5×5 depthwise | 2 | FLOPs 约为 3×3 的 2.8× |
| **标准卷积** | 1×1 pointwise | 1 | NPU 最高效的算子 |
| **注意力模块** | SE (use_se=1) | 3 | GlobalAvgPool + 2×FC，额外归约；⚠️ NPU 可能回退 CPU |
| **扩展比** | expand=3/6 | 3/6 | hidden 通道数线性增加，memory bandwidth 压力 |
| **通道数** | channels（4 档/stage）| 按档位 | 输出通道数，直接影响计算量 |
| **层数深度** | n_blocks=1/2/3 | 1/2/3 | 深度线性增加 |

**架构级硬件成本聚合**（仅累加可搜索项）：

$$C_{total}(A) = \sum_{s \in \text{生效 stages}} n\_blocks_s \cdot \left( C(kernel_s) + C(expand_s) + C(use\_se_s) + C(channels_s) \right)$$

用途：① 种群初始化偏置；② 变异方向偏置；③ 前沿可视化辅助轴。

> 成本表为手工先验，不替代实测。阶段2 延迟数据集到位后，用 Spearman 相关校验 $C_{total}$ 与 RK3566 实测延迟的一致性（目标 > 0.8）；相关性结果写入论文作为成本表合理性的证据，必要时按实测回标各算子权重。

### 搜索空间局限与后续改进

当前空间为标准 MBConv，**不主张结构新颖性**（贡献在搜索方法 + 真机部署 + 新域）。读代码确认的局限及处理：

| 局限 | 影响 | 处理 |
|---|---|---|
| ~~输入分辨率固定 224~~ → **已纳入协同搜索** | 见下"输入分辨率协同搜索"（已实现+已验证）| ✅ 第 17 维 resolution∈{64..224}，第二目标改 MACs 使其可感知 |
| **SE 块（use_se）** | RKNN 上 GlobalAvgPool+FC+Hardsigmoid 可能回退 CPU，延迟跳变 | 板子到位后验证算子支持；若总回退则从空间删除 use_se |
| **depthwise 卷积 NPU 利用率** | NPU 为稠密卷积优化，深度可分离算术强度低，参数/FLOPs 不代表 NPU 延迟 | 再次印证第二目标须换真机延迟 |
| **下采样深度仍最多 /16** | 早期层长期高分辨率；分辨率协同搜索已部分缓解 | 可选后续：加第 4 个 stride-2 让网络能到 /32 |

#### 输入分辨率协同搜索（已实现 + 已验证）

**动机**：输入分辨率是边缘延迟最大的杠杆（MACs ∝ H×W）。固定单一分辨率（如 224 或 128）会漏掉精度-延迟前沿上"又快又准"的点，且**最优分辨率与网络容量耦合**——大网络喂高分辨率才用得上，小网络喂高分辨率是浪费。故把分辨率作为第 17 维与结构**协同搜索**（NSGANetV2/MCUNet 同思路）。

**前提（关键）**：参数量**不随分辨率变**，用参数量作第二目标时搜索看不见分辨率代价、会恒选 224 → 协同搜索失效。故第二目标改为 **MACs**（∝ 分辨率，且比参数量贴近延迟；板子到位后再换真机延迟，仅改 `compute_cost` 一处）。

**分辨率敏感性（MobileNetV2 0.5×，`resolution_sweep.py`，固定结构、手动变分辨率）**：

| 分辨率 | MACs | 相对 224 | Test Acc | Δacc |
|---|---|---|---|---|
| 224 | 95.9M | 1.0× | 98.58% | 基准 |
| **128** | 31.3M | **0.33×** | 97.77% | **−0.81%** |
| 96 | 17.7M | 0.18× | 96.56% | −2.02% |
| 64 | 7.9M | 0.08× | 96.02% | −2.56% |

> 224→128 省 3× 算力仅掉 0.8%（甜点 128），但再降代价上升——是真实 tradeoff，正说明该交给搜索按预算决定，而非写死。

**协同搜索验证（full, seed1, pop20/gen15, MACs 目标）**：最终 Pareto 11 个架构上，**网络参数量与搜索选定分辨率呈强正相关 r = 0.736**：

| 选定分辨率 | 架构数 | 参数量均值（容量）|
|---|---|---|
| 96（大）| 4 | 0.808M |
| 64（小）| 7 | 0.326M |

> **结论**：算法自发学会"**按网络容量匹配分辨率**"——大容量网络配高分辨率（用得上），小网络配低分辨率（省算力）。这①证明固定单一分辨率（128）对小网络不适配，应交给搜索；②是相对 NSGANetV2/MCUNet 的一个 EDA 视角增量——学习 P(分辨率 | 容量) 的条件偏好。
> 说明：此为单 seed、proxy 估计值（仅排序用）、MACs 目标下的**机制验证**；终版需 ≥3 seed + 真机延迟目标重跑坐实。

## 基线模型配置

> **对比框架（重要）**：本任务准确率已接近饱和（实测 Manual-CNN 98.4%、MobileNetV2 98.8%），
> 几乎所有基线都会落在 97–99%，**纯准确率对比无区分度**。故基线对比的核心放在**效率轴**：
> 在 Acc ≥ 97% 的等准确率约束下，比较 RK3566 实测延迟、参数量、FLOPs。
> 搜索空间参数量实测范围 0.015–3.66M（中位 0.20M），下列基线预算均落在可达区间内。
>
> **统一口径**：所有基线一律同一管线**从零训练**（SpecAugment、相同 epoch、≥3 seed 取均值），
> 不使用 ImageNet 预训练，以保证与 NAS 搜出的架构同口径可比。

### 核心基线（必做）

| 类别 | 模型 | 参数量 | 作用 |
|------|------|--------|------|
| 手工下限 | Manual-CNN | ~0.5M | 4层CNN+2层FC，sanity 下限：搜索结果应在效率轴上支配它 |
| 同范式关键 | MobileNetV2 0.5× | ~1.4M | **最重要基线**：搜索空间即 MobileNetV2 范式，直接"手工 vs 搜索"同范式对照 |
| 异范式 | ShuffleNetV2 0.5× | ~1.4M | Channel shuffle 范式，验证跨范式优势 |
| 精度上限参照 | MobileNetV3-Small | ~2.5M | 行业标准，作"大模型精度天花板"参照（NAS 以 ~1/5 参数逼近其精度即为成功）|

### 搜索方法基线（必做）

| 基线 | 说明 |
|------|------|
| Random Search ×200 | 随机采样 200 个取最优，验证进化引导的必要性 |
| Standard NSGA-II | 即消融的 `baseline` 组（关 EDA + Surrogate + 智能初始化），验证三组件的整体增益 |

### 选做基线（视时间/价值取舍）

| 模型 | 参数量 | 取舍理由 |
|------|--------|---------|
| Manual-CRNN（CNN+GRU）| ~0.3M | 佐证"边缘部署不该用 RNN"。⚠️ **GRU 在 RK3566 NPU 常不支持、回退 CPU，延迟差**——这恰是支持纯 CNN 搜索空间的论据，但须在论文显式标注该部署 caveat，否则其延迟会"莫名其妙地差" |
| GhostNet 0.5× | ~1.3M | 异范式补充；饱和任务下边际价值低，时间紧可略 |
| EfficientNet-Lite0 | ~2M | **替代原 EfficientNet-B0 0.5×**：B0 本身是 NAS 产物（拿它当基线混淆"你的搜索 vs 它的搜索"）、无标准 0.5× 缩放、torchvision 不直供；改用专为边缘设计的 Lite0 更干净，或直接删 |

## 实验设计 —— 围绕 4 个 Research Questions 组织

### RQ1：搜索出的架构在「精度–效率」权衡上是否优于手工设计基线？

> 本任务准确率饱和（基线普遍 97–99%），故 RQ1 **不以"准确率更高"为主张**，
> 而以**等准确率下效率更优 / Pareto 支配**为主张。

| 对比项 | 方法 | 报告指标 |
|---|---|---|
| 核心手工基线 | Manual-CNN, MobileNetV2 0.5×, ShuffleNetV2 0.5×, MobileNetV3-Small（选做：Manual-CRNN, GhostNet 0.5×, EfficientNet-Lite0）| Top-1 Acc, RK3566 实测 Latency, Params, FLOPs |
| 随机搜索 | 200 个随机架构，报告最优 5 个的均值±std | 同上 + HV vs 评估预算曲线 |
| Ours（Full：EDA+Surrogate+分层初始化） | 50 代 × 40 个体，独立重复 5 次 | 同上 + Pareto 前沿图 |

**主结果图**：Acc–Latency 散点 + Pareto 前沿——手工基线为散点、Ours 为前沿曲线，
展示"同精度下 Ours 延迟/参数更低（基线点被前沿支配）"。
**判定标准**：① Ours 前沿支配各手工基线点；② 在 Acc ≥ 97% 处，Ours 最低延迟 < 最优基线延迟。

#### 与当前 SOTA NAS 方法对比（投一区必需）

只跟手工基线和随机/标准 NSGA-II 比不足以支撑一区——必须证明相对**当前最强 NAS 方法**的竞争力。

| 类别 | 代表方法 | 对比方式 |
|---|---|---|
| 进化多目标 NAS | **NSGA-NetV2**（Lu et al., ECCV'20）| 同搜索空间/数据/预算重跑，比 HV、C-metric、Pareto 图（与本方案同范式，最 apples-to-apples 的方法级对比）|
| 硬件感知可微 NAS | **ProxylessNAS**（Cai et al., ICLR'19）| 公开代码，以 RK3566 实测延迟为目标在本数据上重搜 |
| 高效超网 NAS | **Once-for-All**（Cai et al., ICLR'20）| 用其超网在 RK3566 延迟约束下检索子网，retrain 后比 |
| 边缘极小模型 NAS | **MCUNet / TinyNAS**（Lin et al., NeurIPS'20）| 迁移其搜索流程/已发布架构到本数据，retrain + 上板测延迟（直接对标"极小模型"卖点）|

**公平协议**：所有方法统一 ① 同数据集与划分；② 同训练配方（SpecAugment、30ep、同优化器）；③ 同 RK3566 实测延迟口径；④ 同等搜索预算（真实评估次数 / GPU·小时对齐）。报告 HV、C-metric(Coverage)、Pareto 前沿叠加图、Wilcoxon 检验。

> **优先级**：NSGA-NetV2 是方法级核心对手（必做）；ProxylessNAS/OFA/MCUNet 至少做 1-2 个代表（硬件感知 SOTA）。全部依赖 RK3566 实测延迟，需板子到位。

##### 核心对手专述：NSGANetV2（Lu et al., ECCV 2020）

NSGANetV2 = **NSGA-II + 代理辅助 + MobileNet(MBConv) 搜索空间 + 精度 vs 硬件延迟**，是 NSGA-Net（GECCO'19）的升级版。它的代理是双重的：一个**在线精度预测器**（搜索中不断 refine，并在 RBF/MLP/GP 等多种预测器间**自适应选最准的**），外加一个 **supernet（权重共享）** 提供热启动权重做廉价评估。这与本方案的骨架**高度重合**，是最 apples-to-apples、也最需要正面交代的对手。

**相似点（坦诚列出）：**

| 维度 | NSGANetV2 | 本方案 |
|---|---|---|
| 进化主干 | NSGA-II | NSGA-II（相同）|
| 代理辅助 | 在线精度预测器（搜索中 refine）| RF + 每代 infill 回填（同思路）|

### RQ5（选做）：Post-NAS 结构化剪枝对最优架构的进一步压缩

NAS 决定宏观架构（层数、宽度、算子），剪枝在此基础上删除训练后的冗余权重，属于不同阶段的正交压缩手段，可叠加使用。

**动机**：NuWa（CVPR 2025）在 Jetson Orin NX（同类 ARM 边缘板）上实验证明，INT8 量化因缺乏 TensorRT 支持吞吐量下降 98%+，非结构化稀疏剪枝几乎无加速——结构化剪枝是边缘部署唯一实用的后处理压缩手段。NAS 搜出的最优架构若参数量仍偏高（1M–2M），可通过剪枝进一步逼近 <500K 的理想目标。

**方法**：对 NAS 最优架构的 pointwise（1×1）卷积层执行激活引导结构化剪枝：

$$W_2' = W_2 \mathcal{H}^\top \mathcal{H}_r (\mathcal{H}_r^\top \mathcal{H}_r)^\dagger$$

其中 $\mathcal{H}_r$ 为激活均值最高的 $e'$ 个神经元对应的特征矩阵（128 张校准样本），闭合最小二乘解重建剩余权重，**无需 retrain**。

**实验设计**：

| 对比组 | 说明 | 报告指标 |
|---|---|---|
| NAS-best | Pareto 前沿最优架构（未剪枝） | Acc, Latency, Params |
| NAS-best + Pruning (α=0.2) | 剪枝率 20%，无重训 | 同上 + 精度损失 |
| NAS-best + Pruning (α=0.4) | 剪枝率 40%，无重训 | 同上 + 精度损失 |
| NAS-best + Pruning + Fine-tune | 剪枝后 10 epoch 微调 | 同上 |

若 RQ4 蒸馏已做，则额外对比：NAS → 蒸馏 → 剪枝 的三段式流程（蒸馏提精度上限，剪枝再降体积）。

> 条件不具备时可推迟，不影响论文主体。开销：剪枝本身仅需分钟级（闭合解），Fine-tune 约 1–2 GPU·小时。

## 算法改进：EDA + Surrogate 融合搜索

### 与最接近工作的区别（PMB-CAS）

最接近的工作是 PMB-CAS（Sun et al., "Multi-Objective Neural Architecture Search for Cognitive Diagnosis," IEEE TCSS, 2026）。本方案的 EDA 概率建模、NSGA-II 选精英、算子加权成本表、温度退火采样、Laplace 平滑、4-RQ 框架均借鉴该范式——**论文中相关公式（温度采样、条件概率）与框架须显式引用 PMB-CAS**，否则与原文雷同会被审稿人识别。在此之上，本方案可主张的贡献（delta）：

| 贡献 | 相对 PMB-CAS 的区别 |
|---|---|
| ① 新问题域 + 真机硬件目标 | PMB-CAS 是表格认知诊断，第二目标是从架构免费算出的拓扑可解释性；本方案是边缘鸟鸣识别，第二目标为 **RK3566 真机实测延迟**（需 RKNN-Toolkit2 上板 + 延迟预测器），属硬件感知 NAS 这一不同问题类 |
| ② **代理引导的架构感知 EDA**（见组件二）| 一个贡献、两个机制：(a) **架构感知条件结构**——PMB-CAS 对变长计算树建树文法 P(type\|parent,pos,depth)，本方案在定长 17 维向量上依架构语义（n_stages 门控 + stage 内 channels×expand 联合）自定义依赖；(b) **代理引导**——PMB-CAS 明确**不用** surrogate、视 EDA 与代理为对立两路，本方案让代理**反向塑造** EDA 分布（质量加权精英 + 特征重要度自适应每基因温度），构成耦合反馈。(b) 是相对 PMB-CAS 最干净的 delta；消融 `no-sgeda` 隔离 (b)、`no-eda` 验证整个算子 |
| ③ 硬件实测校准的成本表 | PMB-CAS 的 C(op) 是主观可解释性权重、无物理校准；本方案 C(op) 用真机延迟回标（Spearman > 0.8）|

> 定位：**应用新域 + 范式融合**，不主张"全新算法"。过度宣称 EDA 本身的新颖性会因与 PMB-CAS 雷同而被拒。

### 动机：标准 NSGA-II 的两个瓶颈

```
瓶颈 1：评估太慢
  → 每代 40 个体 × 完整训练 50 epoch = 极其昂贵
  → 解决：Surrogate 代理模型替代完整训练

瓶颈 2：生成太盲
  → 交叉+变异是「无记忆」的随机扰动，好的结构模式容易被破坏
  → 解决：EDA 从精英种群学习分布，定向采样
```

### 整体流水线

```
                 ┌─── 代理层 1：评估（Surrogate）──────────┐
                 │  精度：在线随机森林（编码→proxy acc）     │
                 │  延迟：离线延迟预测器（编码→ms，板子到位） │
                 │   【当前：MACs(感知分辨率)；待板换真机延迟】│
                 └────────────────────────────────────────┘
                         ↓ 预测值送入
              ┌──────────────────────┐
              │   NSGA-II 非支配排序   │
              │   + 拥挤距离选择精英   │
              └──────────────────────┘
                         ↓ 精英子集 P_elite（含代理预测精度）
                 ┌─── 代理层 2：生成（EDA）─────────────────┐
                 │  从 P_elite 学习架构层级条件分布 M_prob    │
                 │  + SG-EDA：代理质量加权 + 重要度温度        │
                 │  采样新个体（+ 变异保持多样性）            │
                 └────────────────────────────────────────┘
                         ↓ 形成下一代种群
              ┌──────────────────────────────┐
              │  每代 infill 回填             │
              │  真实评估预测最优 N_INFILL=3 个 │
              │  → 重新拟合随机森林            │
              └──────────────────────────────┘
```

### 算法伪代码

```
Algorithm: EDA-Surrogate NSGA-II for Edge NAS

输入: 搜索空间 S, 种群大小 N, 代数 T_max, 每代回填数 N_INFILL
输出: 最终 Pareto 前沿

1. 初始化种群 P_0: 硬件感知分层初始化 (N 个个体)
2. 评估 P_0: 全部 proxy_eval 真实评估 → 随机森林初始训练集
3. for t = 1 to T_max:
4.     // ① 拟合代理（用全部已积累 proxy 样本）
5.     RF.fit(已积累样本)
6.     // ② 精英选择（NSGA-II）
7.     P_elite = NonDominatedSort + CrowdingDistance
8.     // ③ 概率建模（条件 EDA + SG-EDA）
9.     w = 代理质量加权(P_elite);  τ_d = 特征重要度温度(RF)   // 贡献② 部分B
10.    M_prob = BuildCondProbModel(P_elite, w, τ_d)            // 贡献② 部分A，见组件二
11.    offspring = SampleFromModel(M_prob) + 变异
12.    // ④ infill：子代先 RF 预测，最优 N_INFILL 个真实评估，其余用预测值
13.    评估 offspring（infill 回填）→ 加入已积累样本
14.    // ⑤ 环境选择
15.    P_{t+1} = NSGA-II-Select(P_t ∪ offspring, N)
```

### 组件一：Surrogate 代理模型（两段式）

两个目标的代理路径**不对称**，分开设计：精度用**在线代理**（边搜边学），延迟用**离线预测器**（无法精确计算，须建一次数据集）。

#### ① 精度代理：在线随机森林 + infill 回填（已在 `nsga2_eda.py` 实现）

不采用"离线训 300 个完整架构"的静态预测器，改为 **surrogate-in-the-loop** 的在线主动学习——无前期离线开销、且随搜索聚焦到访问区域：

```
gen 0   : 整个种群用 proxy_eval 真实评估（3 轮/10% 数据）→ 代理初始训练集
gen ≥1  : 用「全部已积累 proxy 样本」重新拟合随机森林（每代一次）
          子代先由 RF 预测精度 → 挑预测最高的 N_INFILL=3 个真正 proxy_eval（infill 回填）
          其余直接用预测值 → 代理越搜越准
```

- **代理模型**：随机森林回归（`SurrogateModel`），输入 = 17 维编码 + 4 个衍生特征（n_stages、Σn_blocks、Σchannels、resolution）。
- **两级降本**：proxy_eval 把"单次评估"从完整训练降到 3 轮/10%；RF+infill 再把每代真实评估数从"全部子代"降到 3 个。
- **为何不用离线学习曲线预测器**：① 任务精度饱和（基线普遍 97–99%），代理只需**排序正确**，不需绝对精度；② 离线训 300 个完整架构需数十 GPU·小时，与"省算力"主张相悖；③ 在线主动学习更契合 EA，专注当前前沿区域。
- **代价**：RF 仅用编码、无学习曲线特征，且前几代冷启动——以 gen0 全种群真实评估作种子 + 持续 infill 回填缓解。

> 校准说明：上方伪代码"每 k 代真实评估 top-5"是早期设计；实现采用**每代** infill 回填 N_INFILL=3 个，等价且更平滑。

#### ② 延迟代理：离线延迟预测器（待 RK3566 板子到位）

延迟**无法像参数量那样精确计算**，而在搜索循环里逐个候选上板实测又太慢（单板 + 数百次搜索）——因此延迟**必须**用预测器，这部分不与精度代理冲突，是必需组件：

- **离线数据集**：采样约 500 个架构 → 导出 ONNX → RKNN(INT8) → RK3566 实测延迟（见阶段 2）。
- **预测器**：输入 = 17 维编码（含分辨率）→ 预测延迟（线性回归 / 轻量 MLP，目标 R² > 0.90、MAE < 2ms）。
- **当前占位（已升级）**：第二目标用 **MACs**（`compute_cost` 计算量，在候选自己的分辨率下统计）。MACs 比参数量贴近延迟、且**能感知分辨率**（参数量不随分辨率变，无法支撑分辨率协同搜索）；板子到位后将 `compute_cost` 替换为延迟预测器，**仅改这一处**。

> 现状提醒：当前第二目标为 **MACs**（前沿是"精度 vs MACs"，已感知分辨率）；早期 full/seed42 用的是参数量。换成真机延迟后，依赖第二目标的大规模搜索/消融仍需重跑——但 MACs 是有意义的中间态（精度 vs FLOPs 是标准 NAS 轴），小规模验证现已可做（见"输入分辨率协同搜索"的 r=0.736 验证）。

### 组件二：代理引导的架构感知 EDA（贡献②）

本方案的 EDA 生成算子作为**一个贡献**呈现，含两个正交机制：**部分 A 架构感知条件结构**（建模哪些变量、如何联合）+ **部分 B 代理引导**（代理反向塑造该分布）。概率建模范式沿用 PMB-CAS（须显式引用），但 PMB-CAS 面向变长计算树、且明确不用 surrogate；本方案在定长 17 维向量上自定义依赖结构，并引入代理引导——**部分 B 是相对 PMB-CAS 最干净的 delta**。

#### 部分 A：架构感知条件结构

> PMB-CAS 建树文法 P(type|parent,pos,depth)；本方案编码为**定长 17 维类别向量**，无法套用树文法，故依据**编码的架构层级语义**自定义依赖结构。

建模分两层（已在 `nsga2_eda.py:conditional_eda_sample` 实现）：

**① 门控位（边缘分布）**：n_stages（d0）学独立边缘分布 $P(\text{n\_stages})$，先采它定下生效 stage 数。

**② 每 stage 基因（门控 + stage 内联合）**：对每个**生效**的 stage $s$（$s <$ n_stages）：
- **门控**：该 stage 的基因只从「stage $s$ 生效」的精英里统计（避免 n_stages=2 精英的无效 stage3 基因污染分布）；
- **stage 内耦合**：对 (channels$_s$, expand$_s$) 学**联合分布**，捕获容量耦合（通道×扩张比 = 容量，二者强相关）；其余 (n_blocks$_s$, kernel$_s$, use_se$_s$) 用条件边缘。

条件概率 + Laplace 平滑（α=1.0）；上下文 $c$=（stage $s$ 是否生效）：

$$P(op_i | c) = \frac{\text{count}(op_i, c) + \alpha}{\sum_j \text{count}(op_j, c) + |\mathcal{O}| \cdot \alpha}$$

**采样顺序**：先采 n_stages → 逐生效 stage 采 (channels,expand) 联合 + 其余条件边缘 → 非生效 stage 随机填（decode 忽略）。

> **与 UMDA 的区别**：独立 UMDA 会拆散精英里的搭配模式（如"高 expand+少块"），并用无效 stage3 基因污染分布；条件结构通过门控 + 联合分布保留这些 building block。这是**部分 A** 的作用，属贡献②的实现细节——代码保留 `no-condeda` 开关可做"条件结构 vs UMDA"的可选探查，但不单列贡献、不进主消融表。

> 温度退火采样中**随代数全局退火**的 $\tau(t)$（沿用 PMB-CAS）当前未启用，列为后续；
> SG-EDA（下）引入的是**按基因的重要度温度** $\tau_d$，两者正交，可叠加。

#### 部分 B：代理引导分布估计（SG-EDA，本贡献的核心 delta）

> PMB-CAS 及标准 surrogate-assisted EDA 中，代理只用于**过滤/加速评估**，与分布建模
> 相互独立（流水线式）。SG-EDA 让代理**反向塑造** EDA 的采样分布，把"代理"与"分布模型"
> 从并列两段升级为**耦合反馈回路**——这是贡献②相对 PMB-CAS 最干净的 delta。
> 已在 `nsga2_eda.py:_elite_weights / _gene_temperatures / _apply_tau` 实现，叠加在部分 A 的条件结构之上。

两个机制（均以随机森林代理为信号源）：

**① 代理质量加权**：精英不再等权进直方图，按其（代理预测）精度 min-max 归一到
$[w_{floor}, 1]$（$w_{floor}=0.2$）作为权重——好精英把分布拉得更狠，最差精英仍保留
$w_{floor}$ 权重以维持探索。加权计数下的边缘/联合分布：

$$P(op_i \mid c) = \frac{\sum_{e \in \text{精英}(c)} w_e \cdot \mathbb{1}[op_i] + \alpha}{\sum_{e} w_e + |\mathcal{O}| \cdot \alpha}, \quad w_e = w_{floor} + (1-w_{floor})\cdot\frac{acc_e - acc_{\min}}{acc_{\max}-acc_{\min}}$$

**② 特征重要度温度**：用随机森林的 `feature_importances_` 给每个基因 $d$ 一个采样温度
$\tau_d$——代理认为**重要**的基因 → $\tau_d \to \tau_{\min}$（=0.5，**锐化**分布，利用）；
**次要**基因 → $\tau_d \to \tau_{\max}$（=2.0，**平滑**分布，探索）。温度缩放
$p' \propto p^{1/\tau_d}$ 作用在 ① 得到的分布上：

$$\tau_d = \tau_{\max} - (\tau_{\max}-\tau_{\min})\cdot\frac{I_d - I_{\min}}{I_{\max}-I_{\min}}, \quad I_d = \text{RF 对基因 } d \text{ 的特征重要度}$$

> **直觉**：代理一边告诉 EDA "**哪些精英**更可信"（质量加权），一边告诉它 "**哪些基因**
> 该集中、哪些该探索"（重要度温度）。需 EDA 与 Surrogate 同开方有意义；消融组
> `no-sgeda` 关掉它（退化为普通计数条件 EDA：精英等权、$\tau=1$），用于隔离**部分 B**（代理引导）的增益。

### 组件三：硬件感知种群初始化

已在 `nsga2_eda.py:smart_init` 实现，分两步（分层 + 成本引导）：

1. **分层（按 n_stages）**：保证 n_stages=2 与 n_stages=3 各占约一半，覆盖深/浅结构。
2. **成本引导（覆盖参数量范围）**：每层从过采样池中按参数量等距挑选，确保初始种群在大小两端都有代表（避免初代全挤在某一参数量区间）。

> 对照消融组 `no-init` 为纯随机初始化。
