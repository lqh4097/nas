# 面向边缘硬件感知的多目标进化神经架构搜索研究方案

## 核心问题

RK3566 边缘端算力严重受限（CPU: 4×Cortex-A55, NPU: 0.8 TOPS），如何在精度和推理延迟之间自动找到最优平衡，打破手工架构的瓶颈。

## 量化目标

| 目标 | 最低要求 | 理想值 |
|---|---|---|
| 鸟鸣分类 Top-1 Accuracy | > 80% | > 90% |
| RK3566 单次推理延迟 | < 50ms | < 20ms |
| 模型参数量 | < 2M | < 500K |
| 模型存储大小（INT8量化后） | < 5MB | < 1MB |

**音频增强方法**（在原始音频上操作后重新生成 mel 图）：

| 增强方法 | 模拟场景 |
|---------|---------|
| PitchShift | 不同个体音高差异 |

| BandStopFilter | 特定频段被遮挡或设备频响不平坦 |

每条原始录音生成 4 个增强变体，增强比 ×5（原始 1 + 增强 4）。

**数据划分**（无需移动文件，按索引区分）：

| 集合 | 来源 | 样本数 | 说明 |
|------|------|--------|------|
| train | Mel_Augment_four/train | 132,230 | 含原始 + 4 种增强，用于模型训练 |
| val | Mel_Augment_four/val 随机 50% | 3,306 | 纯原始 mel，训练监控 + NAS 搜索评估 |
| test | Mel_Augment_four/val 剩余 50% | 3,306 | 纯原始 mel，封存，仅用于最终论文指标 |

test 共 3,306 张，每类约 83 张，95% CI 宽度约 ±1.5%，满足论文汇报要求。
val/test 索引由固定随机种子（seed=42）生成，保存至 `data/split_index.json`。


## 搜索空间定义（核心）

### 宏观架构编码

每个个体用一个整数向量表示，共 16 维：

```
编码位置  参数              取值范围            #取值
──────────────────────────────────────────────────
 0      stem_channels      [16, 24, 32]         3
 1      n_stages           [2, 3, 4]            3
 2      stem_stride        [1, 2]               2
 3      n_layers@stage0    [2, 3, 4]            3
 4      n_layers@stage1    [2, 3, 4]            3
 5      n_layers@stage2    [2, 3, 4]            3
 6      kernel@stage0      [3, 5]               2
 7      kernel@stage1      [3, 5]               2
 8      kernel@stage2      [3, 5]               2
 9      expand@stage0      [1, 2, 3, 4]         4
10      expand@stage1      [1, 2, 3, 4]         4
11      expand@stage2      [1, 2, 3, 4]         4
12      use_se@stage0      [0, 1]               2
13      use_se@stage1      [0, 1]               2
14      use_se@stage2      [0, 1]               2
15      activation         [0(relu), 1(hswish)] 2

总搜索空间大小: 3×3×2×3³×2³×4³×2³×2 ≈ 7,077,888 ≈ 707 万
```

### 微观构建块（每个 stage 内的搜索单元）

```python
class SearchBlock:
    """倒残差 + 可选 SE + 可选跳跃连接，MobileNetV2 block 的搜索版本"""
    def __init__(in_ch, out_ch, kernel, expand, use_se, act, stride):
        # 1. 1×1 expand（如果 expand > 1）
        # 2. depthwise conv (kernel × kernel)
        # 3. SE (如果 use_se == 1)
        # 4. 1×1 project (恢复到 out_ch)
        # 5. skip connection (如果 in_ch == out_ch and stride == 1)

# 输入规格（固定，不在搜索空间内）：
# in_channels = 3（RGB），input_size = 224×224
# stem: Conv2d(3, stem_channels, 3×3, stride=stem_stride)
```

### 搜索空间设计理由（面向时频稀疏信号）

```
设计决策                          针对的声学特性
─────────────────────────────────────────────────
kernel 可选 3 或 5                鸟鸣音节需要更大感受野捕获频率邻域
expand 可选 1-4                  时频稀疏 → 不需过宽的中间表示
SE 可选                           不同鸟种的关键判别频带不同（注意力）
skip connection 有条件            保持浅层时频细节的传递
stem_stride 可选 1 或 2           控制早期下采样程度（保留高频细节）
```

### 算子级硬件成本表

为搜索空间中每个算子预定义硬件成本权重 $C(op)$，作为先验引导信号（不替代实测延迟）。

| 算子类别 | 具体算子/选择 | 成本 C(op) | 理由 |
|---------|-------------|-----------|------|
| **轻量卷积** | 3×3 depthwise | 1 | RK3566 NPU 对 3×3 dw 卷积有专项优化 |
| **中等卷积** | 5×5 depthwise | 2 | FLOPs 约为 3×3 的 2.8× |
| **标准卷积** | 1×1 pointwise | 1 | NPU 最高效的算子 |
| **激活函数** | ReLU | 1 | 几乎无硬件开销 |
| | HardSwish | 2 | 比 ReLU 多 ~30% 延迟 |
| **注意力模块** | SE (use_se=1) | 3 | GlobalAvgPool + 2×FC，额外归约操作 |
| **扩展比** | expand=1/2/3/4 | 1/2/3/4 | 通道数线性增加，memory bandwidth 压力 |
| **层数深度** | n_layers=2/3/4 | 1/2/3 | 深度线性增加 |
| **stem 配置** | stem_channels=16/24/32 | 1/2/3 | 输入通道数 |
| **下采样** | stride=1/2 | 1/2 | stride=2 后续层 FLOPs 指数降低（正向） |

**架构级硬件成本聚合**：

$$C_{total}(A) = \sum_{i \in \text{stages}} \left( C(kernel_i) + C(expand_i) + C(use\_se_i) + C(n\_layers_i) \right) + C(stem\_channels) + C(activation)$$

用途：① 种群初始化偏置；② 变异方向偏置；③ 前沿可视化辅助轴。

## 算法改进：EDA + Surrogate 融合搜索

### 动机：标准 NSGA-II 的两个瓶颈

```
瓶颈 1：评估太慢
  → 每代 40 个体 × 完整训练 50 epoch = 极其昂贵
  → 解决：Surrogate 代理模型替代完整训练

瓶颈 2：生成太盲
  → 交叉+变异是「无记忆」的随机扰动，好的结构模式容易被破坏
  → 解决：EDA 从精英种群学习分布，定向采样
```

### 整体流水线

```
                 ┌─── 代理层 1：评估（Surrogate）───┐
                 │  精度预测器 + 延迟预测器           │
                 │  成本：毫秒级 / 个体               │
                 └──────────────────────────────────┘
                         ↓ 预测值送入
              ┌──────────────────────┐
              │   NSGA-II 非支配排序   │
              │   + 拥挤距离选择精英   │
              └──────────────────────┘
                         ↓ 精英子集 P_elite
                 ┌─── 代理层 2：生成（EDA）─────────┐
                 │  从 P_elite 学习概率分布 M_prob    │
                 │  → 温度退火控制探索/利用            │
                 │  采样 N_sample 个新个体             │
                 │  + N_random 随机注入保持多样性      │
                 └──────────────────────────────────┘
                         ↓ 形成下一代种群
              ┌──────────────────────┐
              │   定期校准（每 k 代）  │
              │   真实评估 5 个精英    │
              │   → 增量更新 Surrogate │
              └──────────────────────┘
```

### 算法伪代码

```
Algorithm: EDA-Surrogate NSGA-II for Edge NAS

输入: 搜索空间 S, 种群大小 N, 代数 T_max, 校准间隔 k
输出: 最终 Pareto 前沿

1. 离线: 随机采样 300 个架构，完整训练 → 训练初始 Surrogate
2. 初始化种群 P_0: 硬件感知分层初始化 (N 个个体)
3. for t = 0 to T_max - 1:
4.     // ① 快速评估（Surrogate）
5.     F(A) = ( -SurrAcc(A), SurrLat(A) )   for each A ∈ P_t
6.     // ② 精英选择（NSGA-II）
7.     P_elite = NonDominatedSort + CrowdingDistance
8.     // ③ 概率建模（EDA）
9.     M_prob = BuildProbModel(P_elite)   // 学习各维条件概率分布
10.    // ④ 温度控制采样
11.    τ = max(0.1, 1.0 - 0.9 × t / T_max)
12.    P_{t+1} = SampleFromModel(M_prob, τ, N_sample) + RandomInit(N_random)
13.    // ⑤ 定期校准
14.    if t % k == 0:
15.        真实评估前沿 top-5 → 加入校准集 → 增量更新 Surrogate
```

### 组件一：Surrogate 代理模型

**离线数据**：随机采样 300 个架构完整训练，得到 `{(编码向量, 真实Acc, 真实Lat)}`。

**精度预测器**：输入 = 16 维编码 + 早停 5-epoch accuracy → 预测最终 accuracy（XGBoost 或 2 层 MLP）。

**延迟预测器**：输入 = 16 维编码 → 预测 RK3566 推理延迟（线性回归或轻量 MLP，预期 R² > 0.90）。

**评估路径**：
```
编码向量 → 解码为 PyTorch 模型 → 训练 5 epoch（早停）
    ├→ 早停 acc + 编码 → 精度预测器 → 预测 accuracy
    └→ 编码 → 延迟预测器 → 预测 latency
总耗时：约完整训练的 1/5
```

### 组件二：EDA 概率建模

对 16 维编码按结构依赖关系学习条件概率分布，加 Laplace 平滑（α=1.0）：

$$P(op_i | c) = \frac{\text{count}(op_i, c) + \alpha}{\sum_j \text{count}(op_j, c) + |\mathcal{O}| \cdot \alpha}$$

温度控制采样：

$$p_i' = \frac{p_i^{1/\tau}}{\sum_j p_j^{1/\tau}}, \quad \tau(t) = \max(0.1,\ 1.0 - 0.9 \times t / T_{max})$$

### 组件三：硬件感知种群初始化

初始种群分三块，每块用不同概率偏置生成：

1. **轻量子种群**（1/3）：偏小 stem_channels、低 expand ratio、kernel=3、浅层
2. **重量子种群**（1/3）：偏大 stem_channels、高 expand ratio、深层
3. **中等子种群**（1/3）：完全随机，填补中间地带

### 改进效果预期

| 维度 | 标准 NSGA-II | +Surrogate | +EDA | **Full** |
|------|-------------|------------|------|----------|
| 每代评估耗时 | ~10h | ~2h | ~10h | **~2h** |
| 48h 内可探索代数 | ~5 代 | ~24 代 | ~5 代 | **~24 代** |
| 生成方向性 | 无 | 无 | 有 | **有** |
| 精英模式保留 | 可能破坏 | 可能破坏 | 保留 | **保留** |

## 风险及应对

| 风险 | 概率 | 应对方案 |

| 没有 RK3566 板子 | 高 | 用 nn-Meter 做延迟预测；联系导师采购；树莓派替代 |
