# 文献调研报告：边缘硬件感知多目标进化NAS（鸟鸣识别）

> 调研轴：①多目标进化NAS ②硬件感知/声学边缘NAS + 延迟预测器 ③EDA分布估计算法NAS + Surrogate代理 ④鸟鸣/生物声学边缘识别

## 0. 一句话结论

本选题处于**四个轴的交叉点上，几乎没有正面竞品**——多目标 ENAS、硬件感知 NAS、声学边缘部署各自都很成熟，但「**EDA + 双 Surrogate 融合 + RK3566 真机延迟 + 鸟鸣**」这一组合是空白。

最需警惕的不是"撞车"，而是三类近邻工作会成为审稿人的质询点：

1. **ESC-NAS**（音频边缘 NAS）
2. **Gong 的 surrogate + 蒸馏 MONAS**（IEEE TEVC）
3. **PMB-CAS**（EDA 范式来源，防抄袭红线）

**行动点**：ESC-NAS 必须精读并写进 Related Work 做正面 differentiation。相对它的 delta：
- (a) 第二目标是**真机实测延迟的 Pareto 前沿**，而非单一硬件约束惩罚项；
- (b) **EDA 概率生成**，而非纯进化/可微搜索；
- (c) 细粒度鸟鸣（40 类），而非粗粒度环境声。

## 3. 轴三：EDA（分布估计算法）NAS —— 算法范式来源

除已知的 **PMB-CAS（Sun et al., IEEE TCSS 2026）**，新增可引锚点：

| 文献 | 出处 | 对本方案的价值 |
|---|---|---|
| **PMB-CAS: Multi-Objective NAS for Cognitive Diagnosis: Balancing Accuracy and Interpretability With Probabilistic Models**（Sun et al.）| **IEEE TCSS 2026** | **算法范式来源，防抄袭红线**：温度采样(eq.6)、Laplace 条件概率(eq.5)与本方案雷同，**必须显式引用** |
| **Automatic NAS Based on an Estimation of Distribution Algorithm for Binary Classification of Image Databases**（BUMDA / Boltzmann UMDA 搜 CNN）| **Mathematics 2025**，[mdpi.com/2227-7390/13/4/605](https://www.mdpi.com/2227-7390/13/4/605) | **第二个 EDA-NAS 锚点**：证明"EDA 做 NAS"非 PMB-CAS 孤例，降低范式"小众"风险。引来支撑"EDA 概率建模是 NAS 的一条独立有效路线" |
| Neural Architecture Search by Estimation of Network Structure Distributions | [arXiv:1908.06886](https://arxiv.org/pdf/1908.06886) | EDA-NAS 早期工作，独立层类型概率矩阵——本方案"层级条件分布"的对照（本方案更强：建模 stage 内联合 + 门控）|
| GraphPNAS: Learning Distribution of Good Neural Architectures via Deep Graph Generative Models | [arXiv:2211.15155](https://arxiv.org/pdf/2211.15155) | "学好架构的分布"现代版（深度生成模型）。引来说明：本方案用**显式可解释的条件概率表 + Laplace**，而非黑盒生成模型，更适合小样本 + 可分析 |

**定位提醒**：**EDA 本身不是新颖点**。算法 delta 只在：
- ③ 定长 16 维"架构层级条件"依赖结构（PMB-CAS 用变长树文法，BUMDA 用边缘独立）；
- ② EDA + 双 surrogate 融合。

Mathematics 2025 这篇正好把"EDA-NAS 是成立的范式"坐实，从而把新颖性收窄到"融合 + 域 + 真机校准"，反而更稳。

## 5. 缺口分析与定位建议（写进 Intro 的"差距-贡献"逻辑）

```
已有工作的三个独立成熟区          本方案的交叉空白
─────────────────────────────────────────────────
① surrogate-ENAS (Xue/Gong)  ─┐
② EDA-NAS (PMB-CAS, BUMDA)   ─┼─► 无人将 EDA生成 + 双surrogate评估
③ HW-aware audio NAS(ESC-NAS)─┘    融合，且落到 RK3566 真机延迟Pareto
                                    + 细粒度鸟鸣域 + 成本表真机校准
```

**可主张的 4 个 delta（已被文献验证为"未被占据"）**：

1. **EDA + 双 Surrogate 融合** —— Gong-TEVC2023 只做 surrogate 不做 EDA 生成；PMB-CAS 明确不用 surrogate。两者合一确为空白。
2. **RK3566 真机实测延迟做第二目标** —— ESC-NAS 是硬件约束/单目标，HELP 是延迟预测但非进化多目标 + EDA。
3. **定长 16 维架构层级条件 EDA** —— BUMDA/PMB-CAS 都非此依赖结构。
4. **成本表真机 Spearman 校准** —— 文献中成本表多为 FLOPs 代理或主观权重，真机回标是干净的小贡献。

## 7. 风险提示（文献角度）

1. **审稿人很可能来自 Xue/Gong 圈** → surrogate 相关工作引全、对比公平，勿只字不提其 rank/score predictor 路线。
2. **ESC-NAS 被忽略 = 致命** → 它和本方案太像，漏引会被直接质疑 novelty。务必精读 + 表格对比。
3. **EDA novelty 别过度宣称** → 已有 PMB-CAS + BUMDA 两篇，卖点收窄到"融合 + 真机 + 域"，更稳、更难被攻。
4. **延迟预测器要敢和 HELP 比** → HELP 在 GPU/CPU/Jetson 验证，本方案只在 RK3566 单设备，应主动说明"单设备无需元学习、轻量回归即可 R²>0.90"，把简单当作场景适配而非缺陷。
